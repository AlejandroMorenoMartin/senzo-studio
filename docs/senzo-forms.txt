Technical Specification: Senzo Studio Web Forms V2
This document contains the final copy, structure, and technical requirements for the implementation of the Senzo Studio web forms. All UX/UI friction points and logic errors have been resolved, including recent updates to optimize for serverless deployment.


1. Global Tech Stack & Architectural Guidelines
The forms are to be developed using the following stack. Adherence to these constraints is mandatory to ensure performance and avoid deployment failures.
* Framework: React 19 + Vite 6
* Styling: Tailwind v4
* Language: TypeScript (Strict Mode required to ensure robust conditional logic rendering).
* Animations: Framer Motion (Used for smooth multi-step transitions and the dynamic progress bar).
* Hosting & API Constraint (Vercel): The application is hosted on Vercel (Free Tier). To avoid the strict 4.5 MB serverless payload limit, direct file uploads have been completely removed from the architecture. All file sharing must happen via external links (Google Drive, WeTransfer, Frame.io) pasted into the text areas.

4. Global Security & Integration Requirements
* Form Tracking (Progress): Implement a visual progress bar (Step X of 4) for both forms using Framer Motion to reduce abandonment rates.
* Anti-Spam Protection: Implement reCAPTCHA v3 (invisible) or a hidden Honeypot field. Validate the token before calling the serverless API.
* Data Transmission: Payload must be strictly JSON. No FormData parsing is required since file inputs have been eliminated. Send data to the designated webhook or serverless endpoint via HTTPS.
